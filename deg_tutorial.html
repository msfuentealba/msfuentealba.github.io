Differential Expression Analysis

Finding genes that change with age — young vs old in GTEx whole blood, using a two-sample t-test in R.

This tutorial walks through the simplest honest way to find differentially expressed genes: run a t-test on every gene, correct for multiple testing, and look at the strongest hits. The point is to make the core idea transparent before reaching for specialized tools.

1.The core idea

Every differential-expression method, no matter how sophisticated, is answering one question for each gene: is the difference between the two groups bigger than what noise alone would produce? A two-sample t-test asks exactly that, in a form you already know from introductory statistics. We run it once per gene, get a p-value per gene, then correct for the fact that we ran the test thousands of times.

What a t-test does and doesn't do. A t-test compares the two age groups directly, using only the age label. It cannot hold other variables constant. If sex were imbalanced across the age groups it could leak into the result — which is why this dataset is restricted to a single sex. With sex fixed by design, the t-test is a clean comparison.

2.Inputs

Two tab-separated files in the same folder as the script:

expression_matrix.tsv — the first column is the gene symbol; every other column is a sample holding its expression value (TPM). metadata.tsv — one row per sample, with at least a sample_id and an age_group column whose values are young or old.

3.Packages

The analysis itself is base R. Only the plot needs ggplot2, which we install if it isn't already present.

# ---- packages ----
if (!requireNamespace("ggplot2", quietly = TRUE)) install.packages("ggplot2")
library(ggplot2)

4.Load and align the data

We read both files, move the gene symbols into the matrix row names, and — the step that matters most — reorder the matrix columns to match the metadata row order. If a sample's expression column and its metadata row don't line up, every group label is wrong and the whole analysis is silently meaningless.

# ---- load data ----
expr <- read.delim("expression_matrix.tsv", check.names = FALSE)
meta <- read.delim("metadata.tsv", check.names = FALSE)

rownames(expr) <- expr$gene_name
expr$gene_name <- NULL
expr <- as.matrix(expr[, meta$sample_id])

meta$age_group <- factor(meta$age_group, levels = c("young", "old"))

Setting young as the first factor level makes it the baseline, so a positive fold change later reads as “higher in old.”

5.Log-transform

Raw expression values are heavily right-skewed: a few genes are enormous and most are small. A t-test assumes roughly symmetric data, so we compress the scale with a base-2 log. The + 1 avoids taking the log of zero.

logtpm <- log2(expr + 1)

is_old   <- meta$age_group == "old"
is_young <- meta$age_group == "young"

The two boolean masks pick out the columns of each group; we reuse them for every gene.

6.A t-test for every gene

We walk gene by gene. For each one, the log2 fold change is simply the difference of group means on the log scale (which is log2(old / young)), and the p-value comes straight from t.test. We pre-allocate the result vectors so the loop just fills them in.

# ---- t-test per gene ----
genes  <- rownames(logtpm)
log2FC <- numeric(length(genes))
pval   <- numeric(length(genes))

for (i in seq_along(genes)) {
  g <- logtpm[i, ]
  log2FC[i] <- mean(g[is_old]) - mean(g[is_young])
  pval[i]   <- t.test(g[is_old], g[is_young])$p.value
}

res <- data.frame(gene = genes, log2FC = log2FC, p = pval)

7.Multiple-testing correction

We just ran thousands of tests. Even with no real signal, about 5% would clear p < 0.05 by chance alone — hundreds of false positives. Benjamini–Hochberg adjusts the p-values to control the expected fraction of false discoveries, and we sort so the strongest genes rise to the top.

res$padj <- p.adjust(res$p, method = "bonferroni")
res <- res[order(res$p), ]

write.table(res, "deg_results.tsv", sep = "\t", quote = FALSE, row.names = FALSE)
Reading the output. Each row is a gene with its log2FC (effect size and direction), p (raw p-value), and padj (adjusted). Positive fold change means higher in old; negative means higher in young. Genes worth trusting have a small padj and a sizeable fold change — statistical and biological significance together.

8.Plot the top 10 genes

The most intuitive check is to look at the actual expression values behind the top hits: one boxplot per gene, young versus old, with every sample shown as a point. We build a long-format table one gene at a time, then facet.

# ---- plot top 10 genes ----
top10 <- head(res$gene, 10)

plot_df <- data.frame()
for (gn in top10) {
  plot_df <- rbind(plot_df,
                   data.frame(gene = gn,
                              expr = logtpm[gn, ],
                              age_group = meta$age_group))
}
plot_df$gene <- factor(plot_df$gene, levels = top10)

ggplot(plot_df, aes(x = age_group, y = expr, fill = age_group)) +
  geom_boxplot(outlier.shape = NA, alpha = 0.5) +
  geom_jitter(width = 0.15, size = 0.6, alpha = 0.5) +
  facet_wrap(~ gene, scales = "free_y", ncol = 5) +
  scale_fill_manual(values = c(young = "steelblue", old = "firebrick")) +
  labs(title = "Top 10 genes: expression by age group",
       x = NULL, y = "log2(TPM + 1)") +
  theme_minimal(base_size = 11) +
  theme(legend.position = "none")

Using scales = "free_y" lets each panel scale to its own gene, so low-expression genes aren't flattened by a shared axis. The result is a 2×5 grid where the young/old shift is visible at a glance.

Running from the terminal. In RStudio this plot appears in the Plots pane automatically. If you run the script with Rscript there is no on-screen device, so wrap the final ggplot(...) call in ggsave("top10.png", ...) to save it to a file instead.

9.Volcano plot with labeled top 10

The volcano plot puts every gene on one chart: effect size on the x-axis (log2 fold change) and significance on the y-axis (-log10 of the adjusted p-value). The genes that matter sit in the upper corners — large change and statistically significant. We flag those, draw the two cutoff lines, and label the ten strongest genes with ggrepel, which nudges the text so labels don't collide.

if (!requireNamespace("ggrepel", quietly = TRUE)) install.packages("ggrepel")
library(ggrepel)

# flag significant genes and mark the top 10 for labeling
res$significant <- !is.na(res$padj) & res$padj < 0.05 & abs(res$log2FC) > 1
top10 <- head(res$gene, 10)
res$label <- ifelse(res$gene %in% top10, res$gene, NA)

ggplot(res, aes(x = log2FC, y = -log10(p))) +
  geom_point(aes(color = significant), alpha = 0.5, size = 0.9) +
  scale_color_manual(values = c("FALSE" = "grey70", "TRUE" = "firebrick"),
                     labels = c("not sig", "significant"), name = NULL) +
  geom_vline(xintercept = c(-1, 1), linetype = "dashed", color = "grey40") +
  geom_hline(yintercept = -log10(0.05), linetype = "dashed", color = "grey40") +
  geom_text_repel(aes(label = label), size = 3.2, max.overlaps = Inf,
                  min.segment.length = 0, box.padding = 0.4,
                  segment.color = "grey50", na.rm = TRUE) +
  labs(title = "Volcano plot: old vs young (whole blood)",
       subtitle = "top 10 genes labeled",
       x = "log2 fold change (old / young)",
       y = "-log10(adjusted p-value)") +
  theme_minimal(base_size = 12)
Why ggrepel. Plain geom_text stacks labels right on top of their points, which overlap badly in a dense volcano. geom_text_repel pushes them apart and draws a thin connector back to each point, so all ten stay readable.

10.GO enrichment with clusterProfiler

A list of significant genes is more interpretable as biology: are the differentially expressed genes concentrated in particular pathways or processes? Over-representation analysis (ORA) answers this by testing whether any Gene Ontology term appears among the DEGs more often than expected by chance. We use clusterProfiler::enrichGO, which takes gene symbols directly.

if (!requireNamespace("BiocManager", quietly = TRUE)) install.packages("BiocManager")
for (pkg in c("clusterProfiler", "org.Hs.eg.db", "enrichplot"))
  if (!requireNamespace(pkg, quietly = TRUE)) BiocManager::install(pkg)
library(clusterProfiler)
library(org.Hs.eg.db)
library(enrichplot)

# gene set of interest: the significant DEGs
sig_genes <- res$gene[!is.na(res$padj) & res$padj < 0.05 & abs(res$log2FC) > 1]

# background/universe: every gene we actually tested (NOT the whole genome)
universe_genes <- res$gene

ego <- enrichGO(
  gene          = sig_genes,
  universe      = universe_genes,
  OrgDb         = org.Hs.eg.db,
  keyType       = "SYMBOL",
  ont           = "BP",            # Biological Process
  pAdjustMethod = "BH",
  pvalueCutoff  = 0.05,
  qvalueCutoff  = 0.05,
  readable      = TRUE
)

head(as.data.frame(ego)[, c("Description", "GeneRatio", "p.adjust", "Count")], 15)

# visualize the top enriched terms
dotplot(ego, showCategory = 15) +
  labs(title = "GO Biological Process enrichment")
The universe is not optional. Over-representation asks whether a term is enriched among the DEGs relative to the genes that could have appeared. If you omit universe, clusterProfiler falls back to the whole annotation as background, which inflates significance. Passing the tested genes — the ones that survived the zero-expression filter — is the honest choice.
Symbols may drop. Because we pass gene symbols, any that aren't current HGNC symbols (old aliases, etc.) are silently dropped during mapping, so the number of genes clusterProfiler actually uses can be smaller than length(sig_genes). Switch ont to "MF", "CC", or "ALL" for other ontologies. For a version that uses the full ranked list instead of a hard cutoff, look at gseGO (GSEA).
A teaching example. A t-test on log-expression makes the concept of differential expression transparent; production RNA-seq tools such as DESeq2, edgeR, and limma ask the same question but model raw counts and share information across genes so they remain reliable with only a few samples per group.
